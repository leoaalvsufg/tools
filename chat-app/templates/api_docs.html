POST /api/chat

Envia uma mensagem para o chat IA. O sistema detecta automaticamente se precisa fazer pesquisa web e retorna uma resposta contextualizada.

Parâmetros de Entrada

Parâmetro Tipo Obrigatório Descrição
message string Sim Mensagem do usuário para o chat IA
session_id string Não ID da sessão para manter contexto da conversa
provider string Não Provedor de IA: "openrouter" ou "ollama"

Exemplo de Requisição

curl -X POST http://localhost:8000/api/chat \ -H "Content-Type: application/json" \ -d '{ "message": "Qual é o preço atual do bitcoin?", "session_id": "user-123", "provider": "openrouter" }'

Resposta

200 OK
{ "success": true, "response": "O preço atual do Bitcoin é aproximadamente $43,250 USD...", "search_performed": true, "search_results": [ { "title": "Bitcoin Price Today", "url": "https://example.com", "content": "Bitcoin trading at $43,250..." } ], "session_id": "user-123", "timestamp": "2024-01-15T10:30:00Z" }
POST /api/search

Realiza pesquisa web avançada com análise de conteúdo e filtragem de resultados relevantes.

Parâmetros de Entrada

Parâmetro Tipo Obrigatório Descrição
query string Sim Termo de pesquisa
max_results integer Não Número máximo de resultados (padrão: 5)
include_images boolean Não Incluir imagens nos resultados

Exemplo de Requisição

curl -X POST http://localhost:8000/api/search \ -H "Content-Type: application/json" \ -d '{ "query": "inteligência artificial 2024", "max_results": 10, "include_images": true }'
POST /api/sql/generate

Converte linguagem natural em consultas SQL usando IA. Suporta múltiplos tipos de banco de dados.

Parâmetros de Entrada

Parâmetro Tipo Obrigatório Descrição
query string Sim Descrição em linguagem natural da consulta desejada
connection_id string Sim ID da conexão de banco de dados
session_id string Não ID da sessão para histórico

Exemplo de Requisição

curl -X POST http://localhost:8000/api/sql/generate \ -H "Content-Type: application/json" \ -d '{ "query": "Mostre todos os usuários ativos dos últimos 30 dias", "connection_id": "conn-123", "session_id": "sql-session-456" }'

Resposta

200 OK
{ "success": true, "sql_query": "SELECT * FROM users WHERE status = 'active' AND created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)", "query_id": "query-789", "schema": [ { "name": "users", "columns": [ {"name": "id", "type": "int"}, {"name": "email", "type": "varchar"}, {"name": "status", "type": "varchar"}, {"name": "created_at", "type": "datetime"} ] } ] }
